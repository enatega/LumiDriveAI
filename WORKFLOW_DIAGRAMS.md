## ğŸ“± Platform-Specific Considerations

### iOS
- **Audio Recording**: Use `expo-av` with `Audio.Recording.createAsync()`
- **Permissions**: Add `NSMicrophoneUsageDescription` to `Info.plist`
- **Audio Playback**: Use `Audio.Sound.createAsync()` and `playAsync()`

### Android
- **Audio Recording**: Same as iOS, but check `RECORD_AUDIO` permission
- **Permissions**: Request at runtime using `PermissionsAndroid`
- **Audio Playback**: Same as iOS

### Web
- **Audio Recording**: Use `MediaRecorder` API
- **Audio Playback**: Use HTML5 `<audio>` element or Web Audio API
- **CORS**: Ensure backend CORS is configured for web origin

## ğŸ¯ Key Integration Points

1. **Authentication Token**: Must be available before any API call
2. **Session ID**: Generate once per conversation, reuse for all messages
3. **Streaming**: Handle chunks progressively for better UX
4. **Error Handling**: Always wrap API calls in try-catch
5. **Permissions**: Request microphone permission before recording
6. **State Management**: Keep messages, loading states, and session in state
